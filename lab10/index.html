What is a Decentralized Identifier (DID)?

A DID is a new type of identifier (a URI) that enables verifiable, decentralized digital identity. Unlike traditional identifiers (email, usernames, government IDs), DIDs are designed so their controllers can prove control without relying on a centralized authority.

DID structure (simple)

At a high level, a DID is a text string with three main parts: the DID scheme (did:), the DID method name, and the method-specific identifier.

Example DID 
did:example:123456789abcdefghi

DID Document (what it contains)

A DID resolves to a DID Document that may include public keys, verification methods, and service endpoints that allow others to interact with the DID subject securely.

Example DID Document (JSON) 
{
  "@context": ["https://www.w3.org/ns/did/v1"],
  "id": "did:example:123456789abcdefghi",
  "authentication": [{
    "id": "did:example:123456789abcdefghi#keys-1",
    "type": "Ed25519VerificationKey2020",
    "controller": "did:example:123456789abcdefghi",
    "publicKeyMultibase": "zH3C2AVv..."
  }]
}
