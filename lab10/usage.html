How DIDs are Used — Examples

Example 1: Verifiable Credentials

Organizations issue verifiable credentials (e.g., diplomas) tied to a DID. The subject controls the DID and can present proofs to verifiers without exposing unnecessary personal data.

Issuance flow (simple):
  1. Issuer creates credential and signs it.
  2. Credential is associated with the subject's DID.
  3. Holder presents the credential to a verifier, who checks the signature via the DID Document.

Example 2: Authentication with DID Documents

DID Documents store public keys (verification methods). To authenticate, the DID controller signs a challenge; the verifier fetches the DID Document to check the public key and signature.

// pseudo: verify signature using public key from DID document
1) Resolve did:example:123 -> did document
2) Fetch verificationMethod.publicKeyMultibase
3) Use public key to verify signature on challenge

Example 3: DID Methods & Resolution

A DID method defines how DIDs are created, resolved, and updated (e.g., did:key, did:web, did:ion). A DID resolver tool takes a DID and returns a conforming DID Document.
